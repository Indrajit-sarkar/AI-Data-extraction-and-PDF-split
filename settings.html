üìñ Overview

Universal Data Extractor is an AI-powered document processing application that extracts structured data from various document formats using Azure Document Intelligence and Azure OpenAI services.

The application supports batch processing, parallel extraction, PDF splitting with barcode detection, and cloud storage integration.

‚ö° Features

  • Multi-format Support - Process PDF, DOCX, XLSX, CSV, and EML files
  • AI-Powered Extraction - Uses GPT models for intelligent field extraction
  • Barcode Detection - Automatically detect and split PDFs by barcode
  • Vendor Detection - Identify and group documents by vendor
  • Parallel Processing - Process multiple documents concurrently
  • Cloud Integration - Browse and process files from Azure Blob Storage
  • Document Review - Review and approve/reject extracted data

üìÑ Supported Documents

  • PDF - Invoices, forms, contracts, and scanned documents
  • DOCX - Microsoft Word documents
  • XLSX/XLS - Excel spreadsheets with tabular data
  • CSV - Comma-separated value files
  • EML - Email files with attachments

üîç Document Intelligence Setup

Azure Document Intelligence (formerly Form Recognizer) is used to analyze document structure and extract content.

To configure:

  • Go to Azure Portal ‚Üí Create a Document Intelligence resource
  • Copy the Endpoint URL (e.g., https://your-resource.cognitiveservices.azure.com/)
  • Copy one of the Keys from the Keys and Endpoint section
  • Enter these values in the Document Intelligence settings
  • Click "Verify & Save" to test the connection

Multiple Endpoints: Add multiple endpoints for faster processing through load distribution.

ü§ñ Azure OpenAI Setup

Azure OpenAI provides the GPT models used for intelligent field extraction and document understanding.

To configure:

  • Go to Azure Portal ‚Üí Create an Azure OpenAI resource
  • Deploy a GPT model (e.g., gpt-4o, gpt-4-turbo)
  • Copy the full Endpoint URL including the deployment name
  • Copy the API Key from Keys and Endpoint
  • Optionally specify the API version (default: 2024-08-01-preview)
  • Click "Verify & Save" to test the connection

‚òÅÔ∏è Azure Storage Setup

Azure Blob Storage enables processing of files stored in the cloud.

To configure:

  • Go to Azure Portal ‚Üí Storage Account ‚Üí Access Keys
  • Copy the Connection String
  • Enter the Account Name and Account Key
  • Click "Test Connection" to verify access

üìÅ Output JSON Paths

Configure where extracted JSON data is saved:

  • JSON Files Path - Primary location for all extracted JSON files
  • Accepted Path - Location for approved documents after review
  • Rejected Path - Location for rejected documents after review

If folders don't exist, they will be created automatically when you save.

‚úÇÔ∏è PDF Splitting Configuration

Configure how multi-page PDFs are split into individual documents:

  • Enable PDF Splitting - Turn splitting on/off
  • Split Method:
    • barcode_vendor - Use barcode detection first, then vendor detection
    • barcode - Split only by barcode detection
    • vendor - Split only by vendor/header detection
    • ai - Use AI to determine split points
  • Naming Pattern - Pattern for split file names:
    • {original_name} - Original filename
    • {increment} - Sequential number
    • {barcode} - Detected barcode value
    • {vendor} - Detected vendor name

üîß Troubleshooting

Common Issues:

  • Connection Failed - Verify endpoint URLs don't have trailing slashes
  • Authentication Error - Check that API keys are correct and not expired
  • Barcode Detection Failed - Install required libraries: pip install pdf2image pyzbar pillow
  • Path Not Found - Use full Windows paths (e.g., C:\Users\...)
  • Slow Processing - Add multiple endpoints to distribute load
